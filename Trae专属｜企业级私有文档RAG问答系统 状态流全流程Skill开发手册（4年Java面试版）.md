# Trae专属｜企业级私有文档RAG问答系统 状态流全流程Skill开发手册（4年Java面试版）

## 前置全局规则（必看，Trae全局生效）

### 一、项目基础固定约束（全程不可修改）

1. **开发者职级**：4年Java后端开发，熟悉分布式、全局异常、线程池、限流熔断、企业代码规范，拒绝玩具代码

2. **技术栈锁死**：SpringBoot3\.3\.x \+ Maven多模块 \+ MySQL8\.0 \+ Redis7\.0 \+ MyBatis\-Plus4\.0 \+ Hutool5\.8 \+ SpringDoc3接口文档 \+ 本地Chroma嵌入式向量库 \+ 通义千问OpenAPI\(LLM\+Embedding\) \+ JUnit5 \+ Mockito

3. **环境约束**：全程本地运行、不购买云服务器、不备案域名、不上对象存储、文件本地临时目录存储，后续可无缝迁移Milvus/OSS/Docker

4. **RAG架构约束**：标准进阶RAG链路（文件解析→清洗分段→重叠窗口切片→向量化入库→语义召回→重排序→Prompt组装→LLM应答），解决大模型幻觉

5. **代码规范约束**：阿里Java开发手册、RESTful接口、统一返回体、全局异常、日志分级、参数校验、接口幂等、异步线程池隔离AI调用、Token限流、超时重试熔断

6. **Trae运行规则**：严格按照【状态流顺序执行】，禁止跨步骤开发；每一步完成必须人工核验通过，才可进入下一状态；所有技能存入项目根目录`/.trae/skills/`，Trae自动加载绑定

### 二、全局目录预制（提前建好，Trae识别路径）

```shell
rag-enterprise-doc-qa/
├── .trae/
│   └── skills/ # 存放全流程7个skill.md，Trae专属执行技能
├── docs/ # 项目全交付文档：需求、设计、库表、接口、复盘
├── rag-parent/ # Maven父聚合多模块项目
│   ├── rag-common # 公共模块：工具、异常、枚举、返回体、常量
│   ├── rag-core # AI核心模块：LLM封装、Embedding、向量库、RAG链路
│   ├── rag-business # 业务模块：文档管理、问答管理、用户管理
│   ├── rag-admin # 极简后台接口模块
│   └── rag-demo # 本地启动演示模块
├── temp-file/ # 本地临时文件存储，自动定时清理
└── README.md # 面试专属项目自述文档（含难点、优化、踩坑）
```

### 三、七大闭环状态流（固定顺序，不可逆）

状态0：项目初始化\&全局规则锁定 → 状态1：需求评审\&业务边界定义 → 状态2：架构设计\&技术方案定稿 → 状态3：库表设计\&枚举字典开发 → 状态4：多模块骨架\&基础基建开发 → 状态5：核心RAG业务编码开发 → 状态6：容错优化\&单元测试\&面试包装复盘
