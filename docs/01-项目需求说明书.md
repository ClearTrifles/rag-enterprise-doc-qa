
# 企业级RAG文档问答系统 - 需求说明书

## 1. 项目概述

### 1.1 项目背景
本项目旨在构建一个企业级RAG（Retrieval-Augmented Generation）文档问答系统，支持用户上传文档并进行智能问答。系统基于本地嵌入式向量库，全程不上云，确保数据安全。

### 1.2 核心价值
- **知识沉淀**：将企业文档转化为可查询的知识库
- **智能问答**：基于文档内容进行精准回答，标注引用来源
- **本地部署**：无需外部依赖，数据私有化存储

### 1.3 技术栈约束
- 语言：Java 21 LTS
- 框架：Spring Boot 3.3.x
- 向量库：Chroma（本地嵌入式）
- 大模型：通义千问（DashScope）
- 数据库：MySQL + Redis

## 3. 非功能需求

### 3.1 性能指标
- **并发支持**：支持20人本地并发问答
- **响应时间**：问答接口P95响应时间 < 5秒
- **异步处理**：文件解析异步执行，不阻塞主线程

### 3.2 容错能力
- LLM调用超时自动重试（最多3次）
- 接口失败返回明确错误码和提示信息
- 文件损坏或格式错误返回友好提示
- 向量库连接异常自动重连

### 3.3 业务禁忌
- 无注册登录功能
- 无复杂权限体系
- 无付费模块
- 不上云部署

### 3.4 三大后端核心亮点

#### 3.4.1 幻觉兜底机制
- **核心策略**：当知识库无匹配内容时，明确拒绝回答，绝不编造答案
- **实现方式**：通过相似度阈值（配置为0.75）判断召回结果有效性，低于阈值时返回"知识库中未找到相关信息"
- **价值**：保证回答的可信度和准确性，规避大模型幻觉风险

#### 3.4.2 Token限流控制
- **核心策略**：基于Redis实现单用户调用频率限制
- **实现方式**：采用滑动窗口算法，限制单位时间内API调用次数
- **价值**：防止资源滥用，保障系统稳定性和服务质量

#### 3.4.3 异步处理架构
- **核心策略**：文件解析和向量化采用异步处理模式
- **实现方式**：Spring Boot @Async注解 + 自定义线程池，解耦上传与处理流程
- **价值**：避免大文件上传阻塞接口，提升系统吞吐量和用户体验

## 4. 迭代预留功能

### 4.1 架构升级
- 向量库迁移：从Chroma迁移至Milvus集群
- 存储扩展：对接阿里云OSS实现文件持久化
- 权限体系：引入RBAC角色权限管理

### 4.2 功能增强
- 定时更新：知识库定时增量更新机制
- 多模态支持：支持图片、表格等复杂文档解析
- 多轮对话：支持上下文关联的多轮问答
- 知识图谱：构建文档间关联关系图谱

### 4.3 运维监控
- 日志链路追踪：全链路日志追踪与监控
- 性能指标监控：QPS、响应时间、错误率等指标
- 告警机制：关键指标异常自动告警

### 4.4 预留上云迭代方案

#### 4.4.1 架构升级路径
| 阶段 | 升级内容 | 当前状态 |
| :--- | :--- | :--- |
| 第一阶段 | 向量库迁移：Chroma → Milvus集群 | 待迭代 |
| 第二阶段 | 存储扩展：本地存储 → 阿里云OSS | 待迭代 |
| 第三阶段 | 权限体系：无 → RBAC角色权限管理 | 待迭代 |
| 第四阶段 | 部署方式：单机 → K8s容器化部署 | 待迭代 |

#### 4.4.2 上云技术方案
1. **Milvus向量库**：分布式向量数据库，支持水平扩展，适合大规模数据存储和高并发查询场景
2. **阿里云OSS**：对象存储服务，支持海量文件存储，提供高可用保障和自动容灾能力
3. **RBAC权限**：基于角色的访问控制，支持细粒度权限管理，满足企业级安全需求
4. **Kubernetes**：容器编排平台，支持弹性伸缩、滚动升级和自动化运维

## 5. 数据流向说明

### 5.1 文档上传流程
用户上传 → 文件校验 → 本地存储 → 异步解析 → 文本清洗 → 文档切片 → 向量化 → 向量入库

### 5.2 问答流程
用户提问 → 敏感词检测 → 问题向量化 → 向量召回 → 构建上下文 → LLM调用 → 结果返回
