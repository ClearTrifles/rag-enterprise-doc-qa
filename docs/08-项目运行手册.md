# 企业级RAG文档问答系统 - 运行手册

> 本文档详细说明如何在本地环境中启动和测试RAG文档问答系统。

---

## 📋 前置准备

### 1. 必备软件安装

| 软件 | 版本要求 | 下载地址 |
|:---|:---|:---|
| **JDK** | 21+ | [Oracle JDK](https://www.oracle.com/java/technologies/downloads/) 或 [OpenJDK](https://jdk.java.net/) |
| **Maven** | 3.9+ | [Maven官网](https://maven.apache.org/download.cgi) |
| **MySQL** | 8.0+ | [MySQL官网](https://dev.mysql.com/downloads/mysql/) |
| **Redis** | 7.0+ | [Redis官网](https://redis.io/download/) |
| **ChromaDB** | 0.4.x+ | [ChromaDB官网](https://www.trychroma.com/) 或 Docker镜像 |
| **IntelliJ IDEA** | 2023.3+ | [JetBrains官网](https://www.jetbrains.com/idea/download/) |

### 2. 环境变量配置

**Windows系统配置**：

1. 按 `Win + R`，输入 `sysdm.cpl` 打开系统属性
2. 点击「高级」→「环境变量」
3. 在「系统变量」中添加：

| 变量名 | 值 | 说明 |
|:---|:---|:---|
| `JAVA_HOME` | `C:\Program Files\Java\jdk-21` | JDK安装路径 |
| `MAVEN_HOME` | `C:\Program Files\Apache\maven-3.9.0` | Maven安装路径 |
| `aliQwen-api` | `sk-xxx...` | 通义千问API密钥 |

4. 在「Path」中添加：
   - `%JAVA_HOME%\bin`
   - `%MAVEN_HOME%\bin`

**验证配置**：

打开命令提示符（CMD），执行：
```cmd
java -version
mvn -version
```

## 🚀 IDEA启动指南

### 1. 导入项目

1. 打开IDEA，点击「File」→「Open」
2. 选择项目根目录 `f:\JavaCode\rag-enterprise-doc-qa`
3. 等待IDEA加载项目（首次加载可能需要几分钟）

### 2. 配置Maven

1. 点击「File」→「Settings」（Windows/Linux）或「Preferences」（Mac）
2. 搜索「Maven」→「Build, Execution, Deployment」→「Build Tools」→「Maven」
3. 设置「Maven home directory」为你的Maven安装路径
4. 点击「Apply」→「OK」

### 3. 配置数据库连接

**步骤1：创建MySQL数据库**

打开MySQL客户端（如MySQL Workbench或Navicat），执行：

```sql
CREATE DATABASE IF NOT EXISTS rag_doc_db 
DEFAULT CHARACTER SET utf8mb4 
DEFAULT COLLATE utf8mb4_unicode_ci;

USE rag_doc_db;
```

**步骤2：执行初始化脚本**

在IDEA中打开 `rag-infrastructure/src/main/resources/sql/init_tables.sql`，右键选择「Run」执行SQL脚本。

或者在MySQL中执行：
```cmd
mysql -u root -p rag_doc_db < rag-infrastructure/src/main/resources/sql/init_tables.sql
```

**步骤3：启动ChromaDB向量数据库**

ChromaDB是向量数据库，用于存储文档切片的向量表示，是RAG问答的核心组件。

**方式一：使用Docker启动（推荐）**

```cmd
# 拉取ChromaDB镜像
docker pull chromadb/chroma

# 启动ChromaDB服务（端口8000）
docker run -d -p 8000:8000 --name chromadb chromadb/chroma

# 验证服务是否启动成功
curl http://localhost:8000/api/v1/collections
```

**方式二：使用Python启动**

```cmd
# 安装chromadb（需要Python 3.8+）
pip install chromadb

# 启动ChromaDB服务
chroma run --host 0.0.0.0 --port 8000
```

**方式三：使用Docker Compose（推荐用于开发环境）**

在项目根目录创建 `docker-compose.yml`：

```yaml
version: '3.8'
services:
  chromadb:
    image: chromadb/chroma
    ports:
      - "8000:8000"
    volumes:
      - ./chroma-data:/chroma/chroma
    environment:
      - CHROMA_SERVER_HOST=0.0.0.0
      - CHROMA_SERVER_PORT=8000
```

执行启动：
```cmd
docker-compose up -d
```

**验证ChromaDB连接**：

启动成功后，访问 `http://localhost:8000/docs/` 应可以看到输出的接口信息。

> ⚠️ **注意**：ChromaDB必须先启动，否则问答接口会报错"获取集合ID失败"。

### 4. 修改配置文件

打开 `rag-api/src/main/resources/application-dev.yml`：

```yaml
# 修改数据库连接（根据你的MySQL配置）
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/rag_doc_db?useUnicode=true&characterEncoding=UTF-8&serverTimezone=Asia/Shanghai&useSSL=false&allowPublicKeyRetrieval=true
    username: root           # 你的MySQL用户名
    password: root123456     # 你的MySQL密码

  data:
    redis:
      host: localhost        # Redis主机
      port: 6379             # Redis端口
      password:              # Redis密码（如果有）
```

### 5. 启动应用

**方式一：使用IDEA启动**

1. 在IDEA左侧「Project」面板中找到：
   `rag-api` → `src/main/java` → `com.enterprise.rag.api` → `RagApiApplication.java`
2. 右键点击 `RagApiApplication.java` →「Run RagApiApplication」
3. 等待启动完成，控制台显示：
   ```
   Started RagApiApplication in X.XX seconds (process XXXX)
   ```

**方式二：使用Maven命令启动**

打开IDEA底部的「Terminal」，执行：
```cmd
cd f:\JavaCode\rag-enterprise-doc-qa
mvn spring-boot:run -pl rag-api
```

## 🔍 接口测试指南

### 1. 访问Swagger接口文档

启动成功后，打开浏览器访问：
```
http://localhost:8080/swagger-ui.html
```

### 2. 测试文档上传接口

**步骤1：准备测试文件**

准备一个PDF、TXT或DOCX文件（建议先用TXT文件测试）。

**步骤2：调用上传接口**

在Swagger页面找到 `DocumentController` → `/api/documents` → 点击「Try it out」：

| 参数 | 值 | 说明 |
|:---|:---|:---|
| `documentName` | `测试文档` | 文档名称 |
| `file` | 选择本地文件 | 要上传的文档 |

点击「Execute」，成功响应示例：
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "id": 1,
    "documentName": "测试文档",
    "originalFilename": "test.txt",
    "fileType": "TXT",
    "status": 1,
    "statusDesc": "处理中"
  }
}
```

### 3. 测试文档列表接口

在Swagger页面找到 `/api/documents` GET接口 → 点击「Try it out」→「Execute」：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "list": [
      {
        "id": 1,
        "documentName": "测试文档",
        "fileType": "TXT",
        "status": 1,
        "chunkCount": 5
      }
    ],
    "total": 1
  }
}
```

### 4. 测试智能问答接口

**步骤1：等待文档处理完成**

文档上传后需要等待异步处理（约10-30秒），刷新文档列表查看状态变为「可用」。

**步骤2：调用问答接口**

在Swagger页面找到 `QAController` → `/api/qa/ask` → 点击「Try it out」：

```json
{
  "question": "文档中提到了什么内容？",
  "sessionId": "test-session-001"
}
```

点击「Execute」，成功响应示例：
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "answer": "根据文档内容，提到了...",
    "sources": [
      {
        "documentId": 1,
        "content": "文档片段内容...",
        "similarity": 0.85
      }
    ],
    "status": "SUCCESS",
    "responseTime": 1500
  }
}
```

## 🧪 专项测试用例

### 测试用例1：高频问答缓存

**目的**：验证相同问题第二次请求命中缓存

**操作步骤**：

1. 调用 `/api/qa/ask`，问题：`什么是RAG？`
2. 记录响应时间（首次约1-3秒）
3. 再次调用相同问题
4. 验证响应状态为 `CACHE_HIT`，响应时间<100ms

**预期结果**：
```json
{
  "status": "CACHE_HIT",
  "responseTime": 50
}
```

### 测试用例2：无知识库问答

**目的**：验证无匹配文档时返回兜底提示

**操作步骤**：

1. 确保知识库中没有相关文档
2. 调用 `/api/qa/ask`，问题：`什么是量子计算？`
3. 验证响应状态为 `NO_MATCH`

**预期结果**：
```json
{
  "answer": "无法从知识库中找到相关信息",
  "status": "NO_MATCH"
}
```

### 测试用例3：限流测试

**目的**：验证超过限流阈值时被拒绝

**操作步骤**：

1. 快速连续调用 `/api/qa/ask` 10次以上（限流配置为10次/分钟）
2. 第11次调用时验证返回限流错误

**预期结果**：
```json
{
  "code": 429,
  "message": "请求过于频繁，请稍后重试"
}
```

### 测试用例4：文档删除联动

**目的**：验证删除文档时向量库数据被清理

**操作步骤**：

1. 上传一个测试文档
2. 等待处理完成（状态变为「可用」）
3. 调用 `/api/documents/{id}` DELETE接口删除文档
4. 调用 `/api/documents/{id}` GET接口验证文档不存在

**预期结果**：
```json
{
  "code": 404,
  "message": "文档不存在"
}
```

## ⚠️ 常见问题排查

### 问题1：启动时数据库连接失败

**错误信息**：
```
Failed to obtain JDBC Connection; nested exception is ...
```

**解决方案**：
1. 检查MySQL服务是否启动
2. 检查数据库配置中的用户名和密码是否正确
3. 检查数据库 `rag_doc_db` 是否已创建

### 问题2：Redis连接失败

**错误信息**：
```
Unable to connect to Redis; nested exception is ...
```

**解决方案**：
1. 检查Redis服务是否启动
2. 检查Redis配置中的主机和端口是否正确
3. 如果Redis设置了密码，需要在配置文件中添加密码

### 问题3：文档上传失败

**错误信息**：
```
文件类型不支持
```

**解决方案**：
1. 确保上传的是PDF、TXT或DOCX格式
2. 检查文件大小是否超过100MB限制

### 问题4：问答接口返回超时

**错误信息**：
```
AI服务调用失败
```

**解决方案**：
1. 检查环境变量 `aliQwen-api` 是否正确配置
2. 检查网络是否能访问通义千问API（`https://dashscope.aliyuncs.com`）
3. 等待一段时间后重试（可能是临时限流）
